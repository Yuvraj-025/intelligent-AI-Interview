# VoxHire AI — Core Features

## 1. Voice-Based Interview Questions
The system asks interview questions using generated voice.

This should feel like a real interviewer rather than a chatbot reading text.

---

## 2. Voice-Based User Responses
The user answers through microphone input.

The system records the response and converts it to text for evaluation.

---

## 3. Interview Modes

### Normal Interview Mode
Features:
- realistic pacing
- detailed questions
- follow-up questions
- more conversational interview flow

### Rapid Fire Mode
Features:
- fast and direct questions
- time pressure
- shorter answers expected
- speed-based performance evaluation

---

## 4. Role Selection
Users should be able to choose interview category such as:
- Software Development Engineer
- Frontend Developer
- Backend Developer
- Data Analyst
- Data Scientist
- Machine Learning Engineer
- HR / Behavioral
- Cloud / DevOps

---

## 5. Resume-Aware Interviewing (Later / Optional MVP+)
The system can take a user resume and generate role-relevant questions from:
- projects
- skills
- tools
- experience

---

## 6. Adaptive Question Selection
The next question should depend on:
- previous answer quality
- weak topics
- confidence indicators
- mode type
- difficulty progression

---

## 7. Real-Time Scoring
Each response should be analyzed for:
- semantic correctness
- answer completeness
- filler words
- hesitation
- speaking pace
- structure and clarity

---

## 8. Weakness Mapping
At the end of a session, the system should identify:
- weak topics
- repeated problem areas
- communication weaknesses
- confidence-related weaknesses

---

## 9. Final Interview Report
A final report should include:
- total session score
- topic-wise breakdown
- confidence and fluency notes
- strengths
- weaknesses
- recommendations

---

## 10. Dashboard Experience
Users should be able to:
- start interview sessions
- review previous sessions
- compare performance
- revisit transcripts and reports

---

## 11. Timer and Pressure Logic
Especially in Rapid Fire mode:
- countdown timers
- answer timeout logic
- fast progression rules

---

## 12. Session Transcript Storage
The platform should store:
- asked questions
- user answers
- AI evaluations
- session summaries

This helps with review and progress tracking.
